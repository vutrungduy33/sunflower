# Sprint Backlog（MVP）

> 建议 2 周一个 Sprint。以下按 3 个 Sprint（6 周）组织。

## Sprint 1（基础能力）
- 用户登录（微信 code 换取 session）  
- 用户资料/手机号绑定  
- 房型列表/详情/价格日历  
- 订单创建与支付  
- 后台房型管理  
- 后台订单列表  

## Sprint 2（业务闭环）
- 订单取消/改期/退款  
- 餐饮商品管理与点单  
- 服务订单（接驳/猪槽船）  
- 订单状态流转  
- 内容基础（景点/攻略）  

## Sprint 3（运营与沉淀）
- 评价与游记  
- 优惠券/会员权益  
- 报表与对账  
- 运营配置与内容审核
