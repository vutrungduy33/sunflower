## 人工复核步骤

1. 在仓库根目录执行 `docker compose up -d backend`。
2. 执行 `curl http://127.0.0.1:8080/api/health`，确认返回 `code=0` 且 `data.status=UP`。
3. 执行 `docker exec sunflower-mysql mysql -uroot -proot -e "SELECT version, description, success FROM sunflower.flyway_schema_history ORDER BY installed_rank;"`，确认至少包含 `V1 create core tables`（`docker compose` 的 `prod` profile 仅加载 `common` 迁移）。
4. 执行 `docker compose logs --tail=80 backend`，确认 Flyway 已识别 MySQL（`Database: jdbc:mysql://mysql:3306/sunflower (MySQL 8.0)`）且无 `Unsupported Database`。
5. 执行 `docker compose stop backend mysql` 回收本地容器资源。
